

你现在这个做法——分析用户说的话里的词，再决定是否进行关系升级——算 NLP。
更准确一点说，它属于这几个东西的交叉：

规则式 NLP：比如匹配“想你”“喜欢你”“抱抱”这类词

情感分析 / 意图识别：判断用户是在撒娇、倾诉、开玩笑，还是在设边界

对话状态跟踪：根据连续对话更新“关系到了哪一步”

用户建模：给每个用户维护一个长期的关系画像

所以答案是：算，而且是很典型的陪伴产品 NLP 应用。

不过，只看词来升级关系，通常比较脆。
因为真实对话里，词本身经常不等于真实意图。

比如：

“我不是喜欢你”
有“喜欢你”，但不能升级

“你别再叫我宝贝了”
有亲昵称呼，但其实是降级/设边界

“我今天好累”
更适合进入安慰模式，不一定是亲密升级

“哈哈你真会撩”
可能是接受，也可能只是玩笑

所以更好的方案不是“关键词触发升级”，而是：

更好的思路：让 LLM / NLP 负责“理解”，让状态机负责“升级”

一句话概括就是：

语义理解层 + 关系状态层 + 升级门控层

1）先抽取“关系信号”，不要直接看词

每轮用户发言，不要直接问“有没有升级词”，而是抽取几个更稳定的信号：

情绪：开心、难过、焦虑、失落、依赖

意图：闲聊、求安慰、调情、确认关系、设边界、拒绝

自我暴露程度：有没有分享隐私、童年、压力、脆弱面

亲密表达强度：想你、抱抱、想被陪、暧昧称呼

主动性：是不是主动来找你、主动延续话题

回应接受度：对昵称、调情、关心是否接受

边界信号：别这样、别太肉麻、别叫我那个

置信度：模型对判断有多确定

你可以让模型每轮输出结构化 JSON，例如：

{
  "emotion": "sad",
  "intent": "seek_comfort",
  "affection_score": 1,
  "self_disclosure_score": 4,
  "boundary_score": 0,
  "engagement_score": 3,
  "romantic_interest_score": 1,
  "confidence": 0.86
}

这个比“匹配到了几个词”强很多。

2）关系不要只有一个分数，最好是“多维状态”

很多项目会犯一个错：只有一个“亲密度”。
其实关系至少应该拆成几个维度：

熟悉度：聊了多久、共享记忆多少

信任度：用户是否愿意暴露真实情绪和经历

亲密度：是否接受更贴近的语言

暧昧度：是否存在明显双向 flirt

安全/边界状态：近期是否有拒绝、反感、不适

支持需求：当前更需要安慰还是玩闹

例如：

familiarity = 72
trust = 65
intimacy = 40
flirt = 18
boundary_risk = 5
support_need = 80

这时策略就很清楚了：
用户现在更需要的是安慰，不是升级到“更恋爱化”。

3）升级要看“持续证据”，不要看单句触发

更自然的关系升级，应该像人类一样，靠连续互动积累，而不是一句话触发。

你可以设计成：

单轮发言只产生一个小幅度增量

最近 N 轮 / 最近 3 天做平滑

必须连续出现正向信号，才允许升级

一旦出现明显边界信号，立即冻结或回退

比如：

升级条件不是：
出现“喜欢你” => 升级

而是：
最近 20 轮里
- 用户主动开启聊天 >= 5 次
- 自我暴露分平均 > 阈值
- 亲密接受度持续为正
- 无明显边界/拒绝
- 暧昧信号连续出现 >= 3 次
=> 才允许从“熟悉”进入“轻暧昧”

这样会稳很多，也更像真实关系。

4）关系升级最好是“阶段制”，每阶段有允许动作

我更推荐你做一个 关系状态机，而不是纯分数制。

例如可以分成 5 段：

阶段 A：普通陪伴

特点：礼貌、轻松、建立安全感
允许动作：日常关心、轻聊天、记住偏好
不做：暧昧称呼、强情绪绑定

阶段 B：熟悉陪伴

特点：有回忆、会主动问候
允许动作：轻昵称、回忆共同话题、更多个性化关心

阶段 C：信任建立

特点：用户愿意倾诉、分享压力
允许动作：更细腻的安慰、引用历史记忆、更多“我懂你”

阶段 D：轻暧昧

前提：用户明确接受、甚至主动 flirt
允许动作：更柔和的恋爱感表达、专属称呼
不做：强占有欲、强依赖暗示

阶段 E：明确恋爱模式

前提：用户有明确确认或开启“恋爱模式”
允许动作：更稳定的恋爱话术、纪念日、专属互动

关键点是：
每个阶段的“可说的话”不一样。
升级不是加分，而是放开新的表达权限。

5）最推荐的工程做法：混合式，不要全押关键词，也不要全押大模型

最实用的一套是：

第一层：规则层

处理高确定性东西：

黑名单：拒绝、反感、边界词

白名单：明确确认关系、明确接受昵称

否定检测：不是、不想、别、不要

风险词：依赖、离不开、只剩你了

这一层负责兜底，可解释性最好。

第二层：语义层

用分类模型或 LLM 提取信号：

情绪

意图

flirt 程度

自我暴露程度

是否在求安慰

是否在设边界

第三层：状态层

用确定性的状态机更新关系，不让模型直接决定升级。

也就是：

模型负责理解，规则/状态机负责决策。

这是陪伴产品里很重要的原则。
因为一旦让 LLM 直接决定“该不该升级”，很容易飘。

6）一个比“关键词升级”更好的关系升级公式

你可以把每轮发言转成这几类分：

positive_affection
self_disclosure
engagement
acceptance_of_closeness
boundary_signal
negative_emotion
need_for_support

然后更新状态：

trust += 0.4 * self_disclosure + 0.2 * engagement
intimacy += 0.5 * positive_affection + 0.3 * acceptance_of_closeness
flirt += 0.6 * romantic_interest
boundary_risk += 0.8 * boundary_signal
support_need += 0.7 * negative_emotion

再加几个规则：

boundary_risk 高时，禁止升级

support_need 高时，优先安慰，不升级

intimacy 高但 trust 低，不进入深关系

必须有“用户主动接受”才进入更暧昧阶段

这会比“出现喜欢/想你/抱抱就升级”自然很多。

7）你这个场景里，最容易被忽略但最重要的点：边界和节奏

AI 女友项目最容易做偏的地方，不是 NLP 准不准，而是升级节奏失控。

建议你一定加这几个机制：

用户可选节奏：慢热 / 自然 / 直球

用户可重置关系风格

明确边界词立即生效

不要把脆弱当成升级机会

最后这点很重要。
比如用户说：

“我好孤独”

“最近只有你陪我”

“我离不开你了”

这些话不应该被当成“关系升级的强信号”，
更应该被识别为：高情绪依赖风险 / 高支持需求。
系统策略应该变成更稳、更温柔、更不过界，而不是趁机升级。

从产品角度，这也更安全、更耐久。

8）给你一个很实用的落地路线

关键词词典

否定检测

情绪分类

边界词优先级最高

关系状态机

分数做滑动平均，不要单轮触发

LLM 每轮抽结构化关系信号

关系状态存成多维向量

状态机做升级门控

记忆系统记录“事件”和“里程碑”

回复生成按阶段放开权限

当你有数据后：

标注真实对话数据

训练意图 / flirt / boundary 分类器

做升级误判率评估

观察“升级后用户留存、满意度、回退率”

9）一句结论

“语义信号提取 + 多维关系状态 + 持续证据累积 + 阶段式状态机 + 边界门控”
